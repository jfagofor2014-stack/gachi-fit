# ホームカレンダー 設計書

## ゴール
ホーム画面に月間カレンダーを表示し、トレーニング実施日を一目で分かるようにする。日付をタップするとカレンダーの下にその日の実施記録を展開する。

## 前提・制約
- Phase1-3＋記録改善を踏襲：Vanilla JS（ESモジュール）＋ PWA ＋ IndexedDB、テストは `node --test`
- 新規ストアなし（既存 `workouts`/`sets`/`sensoryLogs`/`places` の読み取りのみ）
- 週は日曜始まり

## 機能詳細

### 1. カレンダー表示（ホーム）
- ホーム画面のカウントダウン直下に月間カレンダーを配置
- ヘッダー：`‹ 2026年6月 ›`（前月・次月ボタン）
- 曜日見出し（日 月 火 水 木 金 土）＋日付グリッド
- **実施日**（その日のワークアウトに1セット以上ある日）をライムのハイライトで表示
- 当日はうっすら枠で示す（実施有無と独立）

### 2. 日タップ → 記録展開
- 日付タップでカレンダー下の詳細エリアにその日の記録を描画：
  - セット一覧：種目・`重量kg × 回数（補助N）`・推定1RM・品質スコア
  - 場所（places 名）・トレーニング時間（`formatMinutes`）・本日の感想（note）— 記録があるもののみ
  - 記録が無い日は「この日の記録はありません」
- 別の日を選ぶと差し替え

### 3. アーキテクチャ
- 純粋ロジック `js/lib/calendar.js`：
  - `buildCalendarWeeks(year, month)` → 週ごとの配列。各セルは `'YYYY-MM-DD'` 文字列または `null`（前後の空白）。`month` は 1-12。日曜始まり
- 表示部品 `js/views/calendar.js`：
  - `renderCalendar(container, { trainedDates, initialDate, onSelect })` — 表示月を内部状態で管理し、月グリッド描画・前後月切替・実施日マーク・日タップで `onSelect(dateStr)` を呼ぶ。`trainedDates` は `Set<string>`（'YYYY-MM-DD'）、`initialDate` は当日 `Date`
- `home.js`：
  - `workouts`/`sets` から実施日 `Set` を算出（あるワークアウトに紐づくセットが1件以上ある日）
  - カレンダーを描画、`onSelect` でその日の詳細を下のコンテナに描画する `renderDayDetail(container, dateStr)` を呼ぶ

## データ
- 実施日判定：`sets` の `workoutId` を持つワークアウトの `date` を集約。セットが1件以上ある `date` を実施日とする
- 新規永続データなし

## エラーハンドリング
- 記録が無い日のタップ：詳細エリアに「この日の記録はありません」
- 月移動で実施日マークも該当月に追従

## テスト
- `calendar.js`：
  - `buildCalendarWeeks(2026, 6)`（2026年6月＝6/1は月曜・30日）で
    - 先頭セル（日曜）が `null`
    - `weeks[0][1] === '2026-06-01'`
    - 非null セル数が 30、末尾が `'2026-06-30'`
    - 各週の長さが 7
- カレンダー描画・月移動・日タップ展開はブラウザで手動確認

## 配信
- Service Worker キャッシュを v7 に更新し、`js/lib/calendar.js`・`js/views/calendar.js` を ASSETS に追加

## スコープ外
- 週/年表示、祝日表示、カレンダーからの記録追加・編集（タップは閲覧のみ）
- 実施日以外の指標（ボリューム等）のカレンダー上表示
