# スーパーセット/ドロップセット記録（サイクルB） 設計書

## ゴール
サイクルAで実装した「種目単位のまとめ保存」の土台の上に、②スーパーセット（複数種目を休憩なしで連続実施）と、関連するドロップセット（同種目・重量を休憩なしで連続実施）の記録方法を追加する。

## 前提・制約
- Phase1-3＋各改善を踏襲：Vanilla JS（ESモジュール）＋ PWA ＋ IndexedDB、テストは `node --test`
- 加算的スキーマ変更（DBバージョン変更なし）。`sets` に任意フィールド `groupId`（string, uid）と `groupType`（`'superset' | 'dropset'`）を追加。両方とも通常セットでは `undefined`（既存データはそのまま動作）
- 既存 `getAll`/`put`/`remove`/`uid`、`createStepper`、`estimate1RM`、`categoryKey` 等に準拠
- 既存の「セット入力」カード（種目選択＋行UI）は通常モードとして温存し、モード切替でスーパーセット/ドロップセット用のUIに差し替える

## スコープ外（次サイクル以降）
- 履歴タブ・振り返りタブでのグループ表示（引き続き個別セット表示）
- Obsidian出力・AIインサイトへのグループ情報の反映
- スーパーセット/ドロップセット中の補助レップ入力（通常モードのみ対応）
- ドロップセットの重量減少バリデーション（チェックしない）

## 機能詳細

### モード切替
- 「セット入力」カードの上部にセグメントボタン（通常／スーパーセット／ドロップセット、既存 `bindSeg` パターンを流用）を追加
- モードに応じて種目選択欄・入力欄・PR表示・部位別ボリュームバーの表示を出し分け：
  - 通常：現状のまま（種目1つ・行UI・PR表示・ボリュームバー表示）
  - スーパーセット：種目選択欄を複数種目セレクタに差し替え、PR表示・ボリュームバーは非表示
  - ドロップセット：種目選択欄は単一のまま、PR表示・ボリュームバーは表示（既存と同じ挙動）、行UIから補助レップのトグルのみ非表示

### スーパーセット入力
- 種目セレクタ：2〜4個（初期2、「＋種目を追加」「−種目を削除」ボタンで増減、末尾を対象にする点は既存の行±ボタンと同じ考え方）
- ラウンド表：縦にラウンド（初期3、最小1、最大6、±ボタンで増減）×横に種目、各セルは重量(0.5kg刻み)・回数のステッパー（補助レップなし）
- 「まとめて記録」で、入力済み（重量>0 かつ 回数>0）のセルだけを `sets`＋`sensoryLogs`（note のみ、既存の共通メモ欄を流用）として保存
- 保存する全セットに共通の新規 `groupId` と `groupType: 'superset'` を付与
- 保存順序：ラウンド→種目の順にループし、既存の `createdAt = base + i` 連番方式で本来の実施順を保持
- 保存後はラウンド表を初期状態（3ラウンド・種目2つ・空）にリセット

### ドロップセット入力
- 入力UIは通常モードの行UIをそのまま流用（種目1つ・最大6行・重量/回数ステッパー）。補助レップのトグルのみ非表示にする（スコープ外のため）
- 保存時、通常モードと同じロジックに加えて `groupId`（新規uid）と `groupType: 'dropset'` を全保存セットに付与
- 重量が単調減少しているかのチェックはしない

### 「本日のセット」表示（`renderToday`）
- 同一 `groupId` を持つ連続するセットを1つの枠にまとめて表示
  - 先頭に `groupType` に応じたラベル（`🔗 スーパーセット` / `🔻 ドロップセット`）
  - 枠内は各セットを「種目名 重量kg × 回数」の順で列挙（スーパーセットは種目名が変わる、ドロップセットは同一種目）
  - 編集・削除ボタンは既存どおりセット単位（枠は表示上のグルーピングのみで、`set-editor.js` 側の変更は不要）
- `groupId` のないセット（通常モード）は現状どおり個別表示

## ファイル構成（新規・変更・削除）
- 変更 `js/views/workout.js`：
  - モード切替セグメント追加
  - スーパーセット用の種目複数選択＋ラウンド表UI・保存ロジック追加
  - ドロップセット用の保存ロジック（`groupId`/`groupType` 付与、補助UI非表示）追加
  - `renderToday` を `groupId` に基づくグルーピング表示に変更
- 新規テストなし（`workout.js` はDOM/ビュー層でユニットテスト対象外の既存方針を踏襲、ブラウザ手動確認で検証）
- 変更 `sw.js`：キャッシュバージョン更新（v13→v14）

## エラーハンドリング
- スーパーセット：全セル空なら「少なくとも1セット入力してください」（既存の通常モードと同一メッセージを流用）
- ドロップセット：既存の行UIのエラーハンドリング（空行のみエラー）をそのまま流用。補助レップUIがないため補助>回数チェックは不要

## テスト
- 純粋関数の新規追加はないため自動テストの追加なし（既存49件は現状維持で通過することを確認）
- ブラウザで以下を手動確認：
  - モード切替でUIが正しく切り替わる
  - スーパーセット：種目±・ラウンド±、まとめ保存、`groupId`/`groupType`付きで複数種目のセットが保存される、保存後リセット
  - ドロップセット：既存行UIで保存、`groupId`/`groupType`が付与される、補助UIが出ない
  - 「本日のセット」でグループがラベル付きの枠にまとまって表示される、編集・削除が個別に機能する
  - 通常モードの既存動作に回帰がないこと
