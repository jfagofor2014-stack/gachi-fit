# 種目登録フォーム整理・記録タブの部位→種目2段選択 設計書

## ゴール
メニュー管理の種目登録フォームで「部位（自由入力）」と「主要部位（プルダウン）」が独立入力で食い違いうる問題を解消し、主要部位＋詳細の2段構成に統一する。あわせて記録タブのメイン種目選択（通常/ドロップセット）を、部位を選んでから該当種目だけを選べる2段選択に変更する。

## 前提・調査結果
- `exercises` ストアは `{id, name, bodyPart, category, cuePresets, setPattern}`。`bodyPart` は自由入力（例：`胸/上部`）、`category` は `BODY_PARTS`（`背中/胸/肩/脚/腕/その他`）からのプルダウン選択で、互いに独立しているため食い違いうる
- `js/lib/volume.js` の `categoryKey(ex)` は `bodyPart` の `/` より前を優先し、無ければ `category`、どちらも無ければ `その他` を返す（部位別ボリューム集計・ホームの部位別最高ボリュームなどが依存）
- `js/lib/exercisePresets.js` の `DEFAULT_EXERCISE_PRESETS` は `{name, bodyPart, category}` で、`bodyPart` は常に `category/詳細` の形式（例：`胸/上部`, `category: '胸'`）
- 記録タブ（`js/views/workout.js`）の通常/ドロップセットの種目選択は現在 `<select id="w-ex">` に全種目がフラットに並ぶ

## 機能詳細

### ③ 種目登録フォームの整理（`js/views/exercises.js`）
- 「部位（細分化可）」（自由入力の `#ex-part`）と「主要部位」（`#ex-cat` セレクト）の2つの独立入力を、「主要部位」（セレクト、先頭に配置）＋「詳細」（任意・自由入力、例：上部）の2段構成に統一する
- 保存時、`bodyPart` は `主要部位 + (詳細があれば '/' + 詳細)` として内部的に組み立てて保存する（`exercises` ストアのフィールド自体は `bodyPart`/`category` のまま変更なし。既存の `categoryKey` や部位別ボリューム集計との互換性を保つ）
- プリセット検索のチップをタップした時、`#ex-cat` に `preset.category` を、詳細欄に `preset.bodyPart` の `/` 以降（無ければ空）をセットする
- 登録済み種目一覧（`renderList`）の部位表示は、`bodyPart`（主要部位＋詳細を含む文字列）のみを表示し、別立ての `category` チップ表示は削除する（重複表示の解消）
- 既存データの移行は行わない（過去に登録した bodyPart と category が食い違っている種目があっても、そのまま動作する。新規登録・編集分から統一フォーマットになる）

### ② 記録タブの部位→種目2段選択（`js/views/workout.js`）
- 対象は通常/ドロップセットのメイン種目選択カード（`#w-ex-card`）のみ。スーパーセットの各スロット選択（`#w-ss-ex-N`）は現状維持（変更しない）
- 種目選択カードの上部に、部位のセグメントボタン（既存の`.seg`スタイル、モード切替と同じ見た目）を追加。表示するのは実際に登録済み種目がある部位のみ（`categoryKey`でグルーピングし、`BODY_PARTS`の並び順→それ以外の値は末尾）
- 部位ボタンをタップすると、その下の `#w-ex` セレクトの中身をその部位の種目だけに絞り込み、先頭の種目を自動選択する。選択が変わったら既存の `refreshPR()`/`refreshVolumeBar()` を呼び直す（現在の `#w-ex` の `change` イベントと同じ更新ロジックを流用）
- 初期表示時は、部位ボタンの先頭（`BODY_PARTS`順で最初に該当種目がある部位）を選択状態にする

## ファイル構成（変更のみ、新規ファイルなし）
- 変更 `js/views/exercises.js`：登録フォームのUI・保存ロジック・プリセットチップ反映・一覧表示を変更
- 変更 `js/views/workout.js`：`#w-ex-card` に部位セグメントボタンを追加し、種目セレクトの絞り込みロジックを追加

## エラーハンドリング
- 主要部位は必須（既存どおりセレクトなので常に値がある）。詳細欄は任意で空文字可
- 記録タブで、選択中の部位に種目が1件もなくなる状況は基本発生しない（表示される部位ボタンは種目が存在するものだけなので）。ただし種目を全削除した場合は既存の「先に「メニュー」で種目を登録してください。」表示にフォールバックする（既存ロジックのまま）

## テスト
- `exercises.js`・`workout.js` はDOM/ビュー層のため既存方針どおりユニットテスト対象外。ブラウザで以下を手動確認：
  - メニュー管理で主要部位「肩」・詳細「前部」を入力して登録→一覧に「肩/前部」と表示される
  - プリセット検索で「ベンチ」をタップ→主要部位に「胸」、詳細に「上部」が自動入力される
  - 記録タブで部位ボタンが登録済み部位分だけ表示される（種目がない部位は出ない）
  - 部位ボタンをタップ→種目セレクトがその部位の種目だけに絞り込まれ、PR表示・部位別ボリュームバーが選択種目に応じて更新される
  - スーパーセットモードの種目選択は従来どおりフラットな全種目リストのままであること（回帰がないこと）

## スコープ外
- スーパーセットの各スロット種目選択への2段選択適用（今回は対象外）
- 既存登録済み種目データの一括マイグレーション（bodyPart/categoryの食い違い解消は新規登録・編集分のみ）
- ホーム画面の挨拶＋部位組み合わせ提案機能（別サイクルで対応）
